# Custom Rule Example

This is an example rule demonstrating how to define custom coding conventions for marketplace plugins.

## Purpose

Rules define behavioral constraints for Claude Code when working with plugins in this marketplace project.

## Example Rule

### Naming Convention

All plugin names must use kebab-case format:

- Good: `code-review`, `test-runner`, `api-docs`
- Bad: `codeReview`, `test_runner`, `APIdocs`

### When This Rule Applies

- When creating new plugins
- When modifying existing plugin manifests
- When updating marketplace.json plugin references

This rule is part of the example-plugin template, showing how to add custom rules to marketplace plugins.
